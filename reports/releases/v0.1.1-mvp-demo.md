# Release v0.1.1-mvp-demo — NetOps Service Manager

## Status

Versão liberada para **demo interna controlada**.

## Resumo

Esta versão entrega o MVP inicial do NetOps Service Manager com foco em inventário local, discovery operacional, análise BGP, auditoria, relatórios e segurança de provisionamento em modo bloqueado por padrão.

## Principais entregas

### Inventário e base local

- Cadastro local de dispositivos.
- Banco PostgreSQL via Docker.
- Estrutura monorepo PNPM.
- API Express + TypeScript.
- Frontend React/Vite.
- OpenAPI + Orval/Zod.

### Discovery

- Discovery persistente por dispositivo.
- SSH como fonte primária.
- SNMP como fallback/complemento.
- Persistência em:
  - discovery_runs
  - discovery_snapshots
  - discovery_evidence
- Evidence sanitizada.
- Recuperação de snapshot após restart.

### BGP

- Lista de peers por categoria:
  - Clientes
  - Operadoras
  - CDN
  - IX
  - iBGP
- Modal de detalhes do peer.
- Prefixos recebidos para clientes.
- Prefixos advertidos para operadoras/CDN/IX.
- Consulta de rotas via SSH real-time.
- Limite máximo de 200 prefixos por página.
- Histórico em bgp_route_history.
- Parser Huawei corrigido para não confundir Update messages com quantidade de prefixos.

### Huawei VRP parser

- BGP verbose.
- Peer description.
- Estado da sessão.
- Uptime.
- Received total routes.
- Received active routes total.
- Advertised total routes.
- Route-policy básico.
- Community-filter/list básico.
- Interface parser.
- L2VPN/VSI/L2VC básico.

### Segurança

- CONFIG_APPLY_ENABLED=false por padrão.
- DRY_RUN_DEFAULT=true por padrão.
- Apply real bloqueado.
- Rollback real bloqueado.
- Senhas não retornam na API.
- SNMP community não retorna no payload público.
- Comandos read-only por allowlist.

### Auditoria e relatórios

- audit_logs.
- Página /audit.
- reports.
- Página /reports.
- Export JSON/Markdown para auditoria.
- Download Markdown para reports.

### Integrações

- integration_settings.
- Página /integrations.
- NetBox readiness-only.
- Sem sync real nesta versão.

### Docker/build

- Dockerfile otimizado.
- BuildKit habilitado.
- pnpm fixado.
- pnpm fetch + install offline.
- .dockerignore limpo.
- Rebuild api/web validado.
